# Local Paper Workflow

## Goal

把用户给的单篇本地论文，整理成一套可写、可插图、可追踪证据的工作目录，而不是直接对着 PDF 生写。

## Recommended Workspace Layout

```text
paper-workdir/
├── source.pdf
├── paper.txt
├── evidence_map.md
├── analysis.md
└── images/
    ├── index.md
    ├── img-000.png
    ├── img-001.png
    └── ...
```

如果已有更成熟的目录结构，可以兼容，但至少要保证有：

- PDF 或等价原文来源
- 一份连续可检索的提取文本
- 一个单独的图片目录
- 图片索引
- 对复杂论文，最好再有一份证据地图草稿

## Material Preparation Order

### 1. 复制原始 PDF

- 保留原始文件，不要直接在用户源目录里修改。
- 在新的工作目录保留一份 `source.pdf` 或原始文件名副本。

### 2. 提取全文文本

优先顺序：

1. `scripts/prepare_local_pdf.py`
2. 系统 PDF 工具，例如 `pdftotext`、`mutool`
3. 可用的 Python PDF 库
4. 用户已有的提取文本
5. 手工拼接的 OCR 或页面截图

如果提取失败：

- 明确记录当前有哪些材料可用
- 继续寻找同篇论文的已提取文本、副本、笔记
- 不要因为缺少一个工具就直接只看标题和摘要

提取完成后，至少先定位：

- 摘要原文
- 章节标题
- 图号 / 表号
- 关键公式
- 关键数字所在页

### 3. 提取图片或页面

优先顺序：

1. 直接提取论文中的图片
2. 如果做不到，渲染关键页面为图片
3. 如果仍做不到，至少保存关键图表截图

目标不是“抽尽一切资源”，而是保证这些视觉证据可被正文使用：

- 方法框架图
- 核心表格或主结果图
- 消融图
- 案例图或可视化图

如果论文没有合适的嵌入图：

- 渲染关键页面
- 截图关键表格
- 在索引里注明“页面截图替代原图”

### 4. 生成图片索引

`images/index.md` 至少记录：

- 文件名
- 来源页或图号
- 建议用途
- 是否计划在正文中实际使用

示例：

```markdown
# Image Index

- `img-004.jpg`: Fig. 2, 方法框架图，可放在“方法框架”部分
- `img-007.jpg`: Fig. 5, full counting 网络图，可放在“大学合著网络实验结果”部分
- `img-008.jpg`: Fig. 6, fractional counting 网络图，可放在“大学合著网络实验结果”部分
```

如果表格没有单独被提取成图片，也要在证据地图中记录表号和关键数字。

## Figure Usage Rules

- 不能只提“见图 3”，必须解释图支持哪条论点。
- 优先使用原论文图，不要无必要地重画。
- 如果论文图片很多，至少确保方法图和结果图都进入成品。
- 如果论文几乎没有图，允许使用页面截图或文本结构图补位。
- 如果论文只有结果图没有方法图，允许把结果图兼作“方法落地示意”，但必须说明原因。

## Reading Strategy After Preparation

材料准备完成后，按这个顺序建证据地图：

1. 标题、摘要、引言：确定问题、动机、主张
2. 方法章节：圈出关键模块、公式、训练推断流程
3. 实验章节：列出实验问题、数据集、基线、指标、关键表图
4. 结论与讨论：提炼局限、未来方向、适用边界
5. 把主要 claim、图表、关键数字写成简短映射表，避免边写边找

如果论文不是标准实验论文，把第 3 步替换成：

- 实证论文：研究设计、变量、样本、关键表图
- 计量 / 历史论文：时间范围、统计对象、分段方式、关键图表
- 综述 / 理论论文：论点结构、证据来源、作者的归纳边界

## Failure Handling

- 没有文本提取工具：改用已有文本或截图，不中断分析
- 没有图片提取工具：改用关键页面截图，不省略图文结构
- 图号不清晰：在正文里注明“依据页面内容判断”
- 表格 OCR 差：给出能确认的数字和趋势，并注明不确定性
- 没有表格图片：在正文或证据地图里手工摘录关键数字

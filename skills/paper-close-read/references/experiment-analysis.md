# Experiment Analysis Guide

## Goal

把结果部分写成“作者想证明什么 -> 如何设计 -> 证据是否足够 -> 结论能说到哪一步”的结构，而不是把表格重新念一遍。

如果论文不是标准实验论文，也按同样逻辑处理实证分析、历史统计、案例比较、分段回归、数据库比较或综述中的证据段落。

## Read Results as Claims

对每个结果小节，先回答：

- 作者想验证的 claim 是什么
- 这个实验设计是否真的对应这个 claim
- 表格或图里最关键的对比是谁和谁
- 最重要的数字是什么
- 这组结果还能不能有别的解释
- 这条结论能精确说到哪一步，不能说到哪一步

## Recommended Breakdown

### 1. 结果目标

- 这是主实验、消融、鲁棒性、效率、案例分析，还是可视化实验
- 它对应哪条核心 claim
- 如果不是实验论文：这是描述性统计、回归结果、历史分段、案例对比，还是数据库比较

### 2. 数据与设置

- 数据集是否覆盖目标场景
- strongest baselines 是谁
- 指标是否匹配任务目标
- 是否存在训练预算、模型规模、数据量不公平
- 如果没有 baseline，要说清最关键的比较维度是什么
- 如果论文有多个数据口径，要说清结果用的是哪一个口径

### 3. 结果读取顺序

优先读取：

1. 最强基线与本文方法的差异
2. 是否跨多个数据集或设置都稳定
3. 是否有 trade-off，例如精度更好但成本显著更高
4. 是否只有个别任务有效
5. 数字最大或最常被作者引用的那几项到底来自哪个图表

### 4. 证据类型如何写

#### 主结果表

- 说清“哪一列最关键”
- 说清绝对提升和相对位置
- 说清是否稳定领先
- 说清关键数字来自哪一行哪一列
- 如果表很大，只摘进入正文所必需的数字

#### 消融实验

- 逐个模块回答“拿掉后发生什么”
- 判断模块是否真的独立贡献了提升
- 留意 ablation 是否同时改变了多个变量

#### 效率或复杂度实验

- 不只写更快或更省
- 写清换来了什么，失去了什么

#### 质性案例或可视化

- 说清案例在支持什么机制
- 不要把案例图当成装饰

#### 历史 / 计量 / 分段分析

- 说清断点、时间段、斜率、增长率或系数在证明什么
- 区分模型识别出的统计现象与作者给出的历史解释
- 判断“阶段划分”是否真由数据支持，还是主要靠事后解释
- 解释最后一段异常趋势时，要先排除窗口效应、老化效应、数据库覆盖变化等替代解释

## Questions for Stronger Empirical Reading

- 作者有没有挑选对自己有利的指标
- 是否只在容易的数据集上领先
- strongest baseline 是否缺席
- 是否把统计显著性、方差、区间信息省略了
- 是否把“相关”写成了“因果”
- 消融是否真的证明该模块不可替代
- 数字是否能支持作者用的措辞强度，例如“dramatic”“substantial”“essential”
- 数据库覆盖、时间窗口或样本构造会不会改变结论

## Output Expectations

结果章节至少要交付这些内容：

- 结果问题
- 数据、基线、指标
- 最关键结果
- 结果意味着什么
- 证据还缺什么
- 结果机制解释
- 至少几条带来源的关键数字
- 对应的图号、表号、页码或公式位置

如果论文的实验是全文最强部分，实验分析的篇幅应当超过背景综述和相关工作。
